배치 처리 시스템
=
큐와 같이 동작하여 먼저 들어온 프로그램이 실행하고, 완료되면 다음 프로그램이 실행된다.

단점
-
1. 짧은 프로그램이 늦게 들어왔다는 이유로 오래 기다려서 실행되고.. 비효율적!
2. 여러 일을 동시에 하고싶다.
3. 여러 사용자가 하나의 컴퓨터를 쓰고싶다.
그렇게 해서,

시분할시스템
=
다중 사용자를 지원하고 컴퓨터 응답시간을 최소화한다.
시간을 분할해서 프로그램을 번갈아가며 실행한다.

멀티 태스킹
=
시분할시스템과 같다. 다만 10~20ms 같이 아주 잘게 시간을 쪼개서 번갈아 실행한다.
그래서, 단일 CPU에서 여러 프로그램이 동시에 실행되는 것 **처럼** 보이도록 한다.

* 멀티태스킹 & 멀티 프로세싱
1. 멀티태스킹 : 단일 CPU
2. 멀티 프로세싱 : 여러 CPU를 하나의 프로그램이 병렬로 실행한다. 실행속도 상승

* 멀티 프로세싱 & 멀티 쓰레드
1. 멀티 프로세싱 : 여러 프로세스가 여러 CPU를 사용. **프로세스 스위칭마다 자원을 초기화해야 하므로 부담** 
	1_1. 대칭 멀티 프로세싱
	1_2. 비대칭 멀티 프로세싱 
2. 멀티 쓰레드 : 단일 프로세스가 여러 쓰레드. Stack을 제외한 메모리를 전부 공유한다.

* 멀티 프로그래밍 & 멀티 프로세싱
1. 멀티 프로그래밍 : 시분할시스템같은 방식으로 운영. 싱글코어 싱글쓰레드에서 여러 동작을 수행해야할 때 지금 프로그램 정지 + 다른 프로그램 실행  
2. 멀티 프로세싱 : 여러 CPU로 여러 프로세스를 동시에 실행

* 멀티 프로그래밍, 멀티 태스킹, 시분할 시스템이 다 비슷한 이야기(프로세스와 태스크의 의미같은 차이 등을 제외하면)

* 멀티 쓰레드가 자원을 공유하면서 어느 한쪽이 불능으로 만들어버린다면 문제.. 그렇다면??
-
Critical section : 커널 오브젝트를 사용하지 않고 동기화한다.
	- 커널 오브젝트 : 운영체제에서 자원을 관리하기 위한 데이터를 저장하는 메모리 블록

* 그럼 어떻게 멀티 프로세싱은 자원을 공유할까?
-
